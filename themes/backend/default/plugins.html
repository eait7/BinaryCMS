Developer Guide: Building Plugins

This CMS actively utilizes the enterprise-grade hashicorp/go-plugin structural architecture entirely. Every single plugin is a separate compiled Linux binary executing natively over local RPC memory pipelines securely!

1. Implement the Go Interface: 
package main

import "github.com/ez8/gocms/pkg/plugin"

type MyPlugin struct{}

func (p *MyPlugin) PluginName() string { return "MyTool" }
func (p *MyPlugin) HookBeforeFrontPageRender(c string) string { return c }
func (p *MyPlugin) HookAdminMenu() []plugin.MenuItem { return nil }
func (p *MyPlugin) HookAdminRoute(route string) string { return "<b>Simple Dashboard Route</b>" }
func (p *MyPlugin) HookDashboardWidget() string { return "<h1>Widget</h1>" }
func (p *MyPlugin) HookAdminTopRightWidget() string { return "" }

func main() {
	plugin.ServePlugin(&MyPlugin{})
}
2. Compile & Upload constraint:

Because these are raw sys.Exec binaries gracefully deployed seamlessly internally natively via hot-reloading structures, you must specifically dynamically compile them matching the destination server architecture! Example: env GOOS=linux GOARCH=amd64 go build -o awesome_plugin .

Hot Reloading Active: Deleting, enabling, or injecting components natively spins interfaces dynamically within 50 milliseconds intrinsically overriding active OS bounds smoothly globally.
