Plugin Developer Guide — User Hooks

Learn how to extend the User Management system with your own plugins

BinaryCMS User Management is built with a plugin-first architecture. Every user-facing surface exposes hooks so your plugins can inject custom UI, store per-user data, and react to user events — all without modifying core code.

Key concept: The user_meta table is your universal extension point. Store any per-user data as key-value pairs without touching the core users schema.

🏗️ Architecture: How Plugins Connect to Users

┌─────────────────────────────────────────────────────────────┐
│ USER REGISTERS → Core creates user in users table │
│ → Calls HookUserRegistered(userID) │
│ → Your plugin initializes its own data │
├─────────────────────────────────────────────────────────────┤
│ ADMIN VIEWS USER → Core renders profile tabs │
│ → Calls HookUserProfileTab(userID) │
│ → Your plugin returns an HTML tab │
├─────────────────────────────────────────────────────────────┤
│ USER VISITS /my-account → Core renders account page │
│ → Calls HookUserAccountCard(userID)│
│ → Your plugin returns an HTML card │
├─────────────────────────────────────────────────────────────┤
│ DATA STORAGE → Plugin reads/writes user_meta table │
│ → Key-value pairs per user, fully automatic │
└─────────────────────────────────────────────────────────────┘

🔔 Hook 1: HookUserRegistered(userID int) string

Called immediately after a new user registers. Use this to initialize plugin-specific data for the user.

func (p *ShoppingPlugin) HookUserRegistered(userID int) string {
    // Create an empty cart for the new user
    db.Exec(`INSERT INTO user_meta (user_id, meta_key, meta_value) 
             VALUES (?, 'shop_cart_items', '[]')`, userID)
    db.Exec(`INSERT INTO user_meta (user_id, meta_key, meta_value) 
             VALUES (?, 'shop_order_count', '0')`, userID)
    return ""
}

📋 Hook 2: HookUserProfileTab(userID int) string

Returns HTML that appears as a tab on the admin user detail page. Use this to show admin-facing data for a specific user.

func (p *ShoppingPlugin) HookUserProfileTab(userID int) string {
    // Query this user's order history
    orders := getOrdersByUser(userID)
    
    html := `<div class="card-header">
        <h3 class="card-title">🛍 Purchase History</h3>
    </div>
    <div class="card-body">
        <table class="table">
            <tr><th>Order ID</th><th>Amount</th><th>Date</th></tr>`
    
    for _, o := range orders {
        html += fmt.Sprintf(`<tr><td>#%d</td><td>$%.2f</td><td>%s</td></tr>`,
            o.ID, o.Total, o.CreatedAt)
    }
    
    html += `</table></div>`
    return html
}

🏠 Hook 3: HookUserAccountCard(userID int) string

Returns HTML that appears as a card on the frontend /my-account page. This is what the user sees on their own account dashboard.

func (p *ShoppingPlugin) HookUserAccountCard(userID int) string {
    orderCount := getMetaValue(userID, "shop_order_count")
    
    return fmt.Sprintf(`<div class="ql-card" style="margin-bottom:2rem;">
        <h3>🛍 My Orders</h3>
        <p>You have placed <strong>%s</strong> orders.</p>
        <a href="/shop/orders" class="ql-btn ql-btn-outline">
            View Order History →
        </a>
    </div>`, orderCount)
}

💾 Using user_meta — The Plugin Extension Table

The user_meta table stores key-value pairs per user. Your plugin can store any data structure here without modifying the core database schema.

Table Structure

CREATE TABLE user_meta (
    id         INTEGER PRIMARY KEY,
    user_id    INTEGER NOT NULL,     -- references users.id
    meta_key   TEXT NOT NULL,        -- your plugin's key name
    meta_value TEXT DEFAULT '',      -- the stored value (string/JSON)
    UNIQUE(user_id, meta_key)       -- one value per key per user
);

Core API Functions

FunctionPurposeExample
models.SetUserMeta(userID, key, value) Insert or update a meta value SetUserMeta(42, "shop_tier", "premium")
models.GetUserMeta(userID, key) Get a single meta value tier := GetUserMeta(42, "shop_tier")
models.GetAllUserMeta(userID) Get all meta as map[string]string all := GetAllUserMeta(42)
models.DeleteUserMeta(userID, key) Remove a meta entry DeleteUserMeta(42, "shop_cart")

Naming Convention

Always prefix meta keys with your plugin name to avoid collisions with other plugins:
shop_cart_id, shop_order_count, shop_stripe_id
hotspot_bandwidth_quota, hotspot_data_used
membership_tier, membership_expires
